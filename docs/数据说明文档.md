# 数据说明文档

## 五大国家公园 · 林草故事地图

---

## 一、数据总览

本项目涉及三类核心数据，分别对应三个 JSON 文件：

| 文件名 | 用途 | 字段数 | 更新频率 |
|--------|------|--------|---------|
| `parks.geojson` | 五大国家公园空间边界 | 5个Feature | 如需替换为精确边界 |
| `parks-content.json` | 公园文本内容、物种信息、图片、参考文献 | 6个子模块/公园 | 内容校对后一次性 |
| `statistics.json` | 面积和物种数量对比数据 | 2组对比 | 数据更新时 |

## 二、parks.geojson — 空间边界数据

### 2.1 数据格式

GeoJSON FeatureCollection，符合 [RFC 7946](https://datatracker.ietf.org/doc/html/rfc7946) 标准。

### 2.2 坐标参考系

**WGS84 (EPSG:4326)**，经纬度坐标。

### 2.3 Feature 属性字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `id` | string | 公园唯一标识 | `"sanjiangyuan"` |
| `name` | string | 公园全称 | `"三江源国家公园"` |
| `nameShort` | string | 简称 | `"三江源"` |
| `order` | number | 西→东排序 | `1` |
| `color` | string | 显示颜色(hex) | `"#06d6a0"` |
| `region` | string | 地理分区 | `"西部"` |
| `title` | string | 称号 | `"中华水塔"` |
| `center` | [number, number] | 中心点经纬度 | `[93.5, 34.5]` |
| `zoom` | number | 默认缩放级别 | `7` |

### 2.4 当前数据精度

目前使用**地理近似多边形**（手工绘制，顶点数 13-28 个），非官方测绘边界。精度足够用于页面展示，但不适用于精确空间分析。

### 2.5 替换方法

将精确边界的 Shapefile 或 GeoJSON 覆盖 `parks.geojson` 即可，需保持 `properties.id` 值与现有数据一致。

## 三、parks-content.json — 内容数据

### 3.1 数据结构

以公园 `id` 为 key 的 JSON 对象，每个公园包含以下子结构：

```
{
  "<parkId>": {
    "name":           string,    // 全称
    "established":    string,    // 设立时间
    "location":       string,    // 地理位置
    "area":           string,    // 面积
    "description":    string,    // 简介概述
    "history": {
      "timeline": [{ "year": string, "event": string }]
    },
    "geography": {
      "topography":   string,    // 地形地貌
      "climate":      string,    // 气候类型
      "hydrology":    string,    // 水文特征
      "ecosystem":    string     // 生态系统
    },
    "species": [{
      "name":         string,    // 中文名
      "latin":        string,    // 拉丁学名
      "level":        string,    // IUCN保护级别
      "description":  string,    // 习性简介
      "baike_url":    string,    // 百度百科链接
      "baike_label":  string     // 链接标签
    }],
    "conservation": {
      "text":         string,    // 保护成效概述
      "highlights":   string[]   // 关键数据要点
    },
    "gallery":        string[],  // 图片URL列表
    "references": [{
      "title":        string,    // 文献/网站名称
      "url":          string     // 链接
    }]
  }
}
```

### 3.2 内容来源

| 内容模块 | 主要来源 | 更新状态 |
|---------|---------|---------|
| 基础信息 | 各国家公园官网、国家林草局 | 待校对 |
| 历史沿革 | 国家公园总体规划、新闻报道 | 待校对 |
| 地理环境 | 地理学文献、公园官网 | 待校对 |
| 旗舰物种 | IUCN红色名录、Wikipedia | 已完成 |
| 保护成效 | 国家公园官方发布数据 | 待更新最新数据 |
| 图片画廊 | 中新网实景照片 | 待补充更多实景 |
| 相关文献 | 各公园官网、联合国教科文组织 | 已完成 |

### 3.3 修改内容的方法

编辑 `data/parks-content.json`，按需修改各字段文本。修改后刷新页面即可看到变化，无需重新部署。

## 四、statistics.json — 对比统计数据

### 4.1 数据结构

```json
{
  "area": [
    { "park": "公园名", "value": 数值, "unit": "单位" }
  ],
  "speciesCount": [
    { "park": "公园名", "value": 数值 }
  ]
}
```

### 4.2 数据来源

- 面积数据：国家林业和草原局官方公布数据
- 物种数量：本项目展示的旗舰物种数量（每公园5种）

## 五、图片数据

### 5.1 当前图源

图片画廊使用**中国新闻网（chinanews.com.cn）**2021年首批国家公园新闻图片，共21张实景照 + 4张主题占位图。

### 5.2 替换方法

1. 将真实图片放入 `images/` 对应子文件夹
2. 更新 `parks-content.json` 中对应公园的 `gallery` 数组，将 URL 改为本地路径（如 `images/sanjiangyuan/01.jpg`）
3. 刷新页面验证

### 5.3 推荐图片参数

- 分辨率：≥ 1200×800
- 比例：16:10 或 16:9
- 格式：JPG/WebP
- 大小：单张 ≤ 500KB（Web优化）

## 八、注意事项

1. **坐标系统一**：所有空间数据必须使用 WGS84 坐标系，如从高德/百度地图提取需转换
2. **字符编码**：所有 JSON 文件使用 UTF-8 编码，不可转为 GBK
3. **图片外链风险**：当前 chinanews.com.cn 外链图片可能失效，建议下载到本地 `images/` 目录
4. **版权声明**：实景图片需标注来源（中新网），统计数据需标注出处
5. **数据版本**：每次修改数据文件后建议在更新记录表中备注
